# Halo Notes 发布排版规范 v1（首版）

> 目标：保证从 X 抓取后的文章可读、可导航、可持续维护。

## 1. 文档头部（必须）
每篇文章必须以如下结构开头：

```md
# 标题

- 原始链接：<...>
- 作者：...（@...）
- 发布时间：...
- X Article：<...>

---
```

## 2. 标题层级（必须）
- 主章节：`##`
- 子章节：`###`
- 禁止出现跳级（例如直接从 `##` 到 `####`）
- 若正文无章节，至少补 2-4 个逻辑分段标题

## 3. 列表与步骤（必须）
- 要点统一使用 `- `
- 连续步骤用有序列表（`1.` `2.`）或 `### 步骤名`
- 不能把整段说明塞进单个列表项

## 4. 命令与代码（必须）
以下内容必须进代码块：
- shell 命令（brew/npm/node/openclaw/curl/git/python 等）
- 配置片段

格式：

```md
```bash
command here
```
```

## 5. 清洗规则（必须）
- 删除乱码替代字符：`U+FFFD`（`�`）
- 合并多余空行（最多保留 1 个空行）
- 去除重复段落（相邻重复）

## 6. 目录与导航兼容（必须）
- 页面目录依赖 `##/###` 自动生成
- 文章长度 > 600 字建议至少 3 个 `##`
- 超短文（无章节价值）可不强行加目录

## 7. 质量分级（首版）
- `S`：API 获取到 `article.plain_text` 的完整正文
- `A`：正文完整但有轻度人工修整
- `B`：仅元数据/预览（待补全文）

## 8. 发布前检查清单（必须）
- [ ] 标题和元信息完整
- [ ] 至少有可导航标题层级
- [ ] 命令都在代码块中
- [ ] 无乱码字符 `�`
- [ ] 页面渲染正常（正文/目录/滚动）

## 9. 后续迭代方向（v2+）
- 自动摘要提炼（3条结论 + 3条行动）
- 自动术语注释
- 自动封面图与阅读时长
- 样式一致性 lint
